## 学习策略建议

**第一遍（快速过）**：理解每道题的思路和模板，不急着自己写，重点建立"模式识别"能力。

**第二遍（动手写）**：遮住代码，只看题目描述，尝试自己写出来。写不出来就看提示，但一定要亲手敲一遍。

**面试时的万能思路**：
1. 看到有序数组 → 双指针 or 二分
2. 看到查找/配对/去重 → 哈希表
3. 看到括号/嵌套/最近 → 栈
4. 看到树 → 递归分解
5. 看到最优值/方案数 → DP
6. 看到排列/组合/所有方案 → 回溯
7. 看到最短路径/层数 → BFS
8. 看到连通区域 → DFS

掌握这些模式后，即使题目变化，你也能快速识别它属于哪种类型，然后套模板解决。祝面试顺利！
