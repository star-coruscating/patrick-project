基于上述4种概念 现在可以来谈谈，**什么是vue响应式的本质**

高度概括性地来讲：vue响应式其实是数据和函数的映射关系 当数据发生变化时 从映射关系表里找出对应的所有函数 然后把这些找出来的函数 全部执行一遍

1. **依赖收集** （收集数据和函数的映射关系）
2. **派发更新**（数据变化后 通过映射关系将所有函数重新执行）

关于第二点 也就是派发更新 其实是很明确的 就是将找到的函数全部重新执行

难点和关键点在于第一点 究竟怎么样才会产生依赖收集？

完整表述:

**当受监控的函数 运行其同步代码时 如果发生了读取响应式数据被拦截 那么就建立了该函数和响应式数据的依赖
一旦建立了依赖关系 后续响应式数据发生了变化 所有依赖的函数 重新执行**


针对于上面这句话 做术语的明确解释

1. 受监控的函数：在vue中指的是watch,watchEffect,组件渲染函数
2. 响应式数据：在vue中指的是ref,reactive,props,computed

当充分理解了vue响应式的本质 特别是理解了上面加粗黄底的完整表述 我们可以再次回头来看一下vue SFC的模板编译



1. 模板的本质其实就是渲染函数 而组件渲染函数 就是==> **受监控的函数**
2. 响应式数据ref在模板中会自动解包


```javascript
<template>
  <div>{{data.a}}</div>
</template>

<script setup lang="ts">
...
const data = ref({
  a: 1
})
</script>
```

相当于访问了 data.value.a

这一步满足了 ==> 读取响应式数据被拦截

1&2 ==> 响应式数据data 和组件渲染函数 建立了依赖关系

所以 后续data发生变化 就会导致组件渲染函数的重新执行 在虚拟DOM以及diff算法的加持之下 页面得到了更新
