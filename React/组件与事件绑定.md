React中可以使用两种方法来声明一个组件

1）类
```javascript
class yourClassName extends React.Component {
  render() {
    return (
      // 一段JSX代码
    )
  }
}
```


2）函数

```javascript
function componentName() {
  return (
    // 一段JSX代码
  )
}
```

早期的函数组件 被称之为无状态组件 一般仅仅用来做纯UI的展示 不会有复杂的逻辑
从React16.8推出hooks以后 现在更多的是使用函数组件了
具体参照后期的hooks章节

-----------------------------------
为组件绑定事件

```javascript
<button onClick={clickHandler}>按钮</button>
```

注意 onClick 必须是驼峰命名

**React的事件处理函数中 传入的事件对象 是 合成事件对象(Synthetic Event)**

如果要访问原生事件对象 可以通过 e.nativeEvent的方式


如果想要取消事件的默认行为 不能直接return false 而是需要显示的调用 e.preventDefault()

上面提到了 有两种方式声明一个组件 类 或者 函数

既然有类 那么自然就会有this指向问题 因此 下面来说下 类组件中的this指向



```javascript
class App extends React.Component {
  handleClick() {
    console.log('点击',this) // this为undefined
  }
  render() {
    return (
      <div>
        <button onClick={this.handleClick}>点击</button>
      </div>
    )
  }
}
```

解决方案 将事件处理函数变成箭头函数
```javascript
//1. 函数声明时 使用箭头函数
handleClick = () => {
  console.log('点击',this)
}

//2. 函数调用时 使用箭头函数
class App extends React.Component {
  handleClick() {
    console.log('点击',this)
  }
  render() {
    return (
      <div>
        <button onClick={() => this.handleClick()}>点击</button>
      </div>
    )
  }
}

//3. 使用bind 指定this的指向
class App extends React.Component {
  constructor(props: PropsWithChildren) {
    super(props)
    this.handleClick = this.handleClick.bind(this)
  }
  handleClick() {
    console.log('点击',this)
  }
  render() {
    return (
      <div>
        <button onClick={this.handleClick}>点击</button>
      </div>
    )
  }
}

```
