组件状态 state 理解它 可以对标 vue中 子组件的数据

早期 类组件称之为有状态组件 函数组件称之为无状态组件 因为状态只能在类组件中设置 (后期hook出现以后 函数组件也支持了 后面再说)

---

在修改状态值的时候 注意不要直接去修改 而是应该通过**setState** 方法

```javascript
this.setState({
//...
})
```

**注意** setState 改变状态 **可能**是异步的

--如果改变状态的代码处于某个HTML元素的事件中 则是异步的 否则是同步的

这么设计的背后的原因是 事件处理函数中可能会多次改变状态的值 如果每次一旦改变了状态的值 就去重新触发render函数的话 开销太大了 所以采用异步的方法 会有一个队列进行跟踪 将多次状态的改变合并为一次

---

如果想要在事件处理函数中拿到最新的状态值 即执行了setstate之后的数据 可以通过如下方法

方法1）核心思想是setstate方法提供了第二个参数 是一个回调函数

```javascript
clickHandle = () => {
    this.setState({
      num: this.state.num + 1
    },()=>{
      console.log('clg in setstate callback',this.state.num);
      
    })
    console.log('clg outside setstate',this.state.num);
    
  }
```

<img width="582" height="465" alt="image" src="https://github.com/user-attachments/assets/2279e513-b07c-4280-8450-d238f695036d" />


方法2）提前使用一个变量才存储计算结果

```javascript
clickHandle = () => {
    const newNum = this.state.num + 1
    this.setState({
      num: newNum
    })
    console.log('clg outside setstate',newNum);
  }
```

---

技巧:

  既然setstate有可能是异步 有可能是同步 所以**永远把setstate当做是异步的** 

   因此也不要在setstate方法后面 直接去访问状态 因为你不确定状态到底更改了没有

   因此你但凡需要使用改变后的状态 一虑通过setstate的第二个参数 即回调函数去拿


   说到这里 但凡牵扯到回调函数 那么就会有**潜在的回调地狱问题**
   
```javascript
clickHandle = () => {
    this.setState({
      num: this.state.num + 1
    },()=>{
      this.setState({
        num: this.state.num + 1
      }, ()=>{
        this.setState({
          num: this.state.num + 1
        })
      })
    })
  }
```
---
为了解决回调地狱的问题 setstate的第一个参数 除了是对象以外 也可以是一个方法

<img width="920" height="347" alt="image" src="https://github.com/user-attachments/assets/7b061428-e72b-4a90-bb60-e27df51b9341" />


```javascript
clickHandle = () => {
    this.setState((prev)=>({num: prev.num + 1}))
    this.setState((prev)=>({num: prev.num + 1}))
    this.setState((prev)=>({num: prev.num + 1}))
  }
```

总结一下：

如果希望在一个事件处理函数中 多次的改变状态 并且排在后面的更改是依赖前面的更改的 此时需要给setstate方法传递函数updater

因为当传递的是函数updater时 React会将updater放进队列中 这样后面的updater中的prev 是前一次updater计算完成后的结果

而如果传递的是对象 每个对象里都在对this.state里的属性做改动 但是setstate本身是异步的 这样的话 后面的setstate拿到的this.state永远都是旧值

```javascript
clickHandle = () => {
    this.setState((prev)=>({num: prev.num + 1}),()=>{
      console.log(this.state.num); // 注意这里打印出的值 是最新的值 即经历了+3以后的值
    })
    this.setState((prev)=>({num: prev.num + 1}))
    this.setState((prev)=>({num: prev.num + 1}))
  }
```



props

和vue几乎一样

函数组件  props直接作为函数的入参
```javascript

function 组件名(props) {
  return {
    // JSX..
    <div>
      <p>{props.name}</p>
      <p>{props.age}</p>
    </div>
  }
}
```

类组件 在constructor中将props通过super传给父类 然后通过this.props来使用(就算没有显示地将constructor写出来 也可以直接在render函数中使用this.props 这是因为如果没有写 在实际运行时 会自动补充一个默认值)

<img width="450" height="274" alt="image" src="https://github.com/user-attachments/assets/131b9441-9763-4d8b-991e-4252155bf142" />

```javascript
class 组件名 extends React.Component {
  constructor(props) {
    super(props);
  }
  render() {
    return (
      <div>
        <p>{this.props.name}</p>
        <p>{this.props.age}</p>
      </div>
    )
  }
}
```


props.children

props.children就是vue中的插槽
```javascript
//... 父组件的render函数 调用了子组件 FunctionComponent
render() {
    return (
      <div>
        <FunctionComponent info={this.state}>this is a function component !!</FunctionComponent>
      </div>
    )
  }



// 子组件
// 在子组件中 就可以通过props.children 拿到 this is a function component !! 这段内容
function FunctionComponent(props) {
    const { info } = props
    return (
        <>
            <div>FunctionComponent {info.num}</div>
            <div>props.children: {props.children}</div>
        </>

    )
}
```

当然了 vue中还有类似于具名插槽 作用域插槽等用法 这种用法在react中都是不支持的


还有一点关于props的知识点

props如果传递的数据是字符串 那么很简单 就像下面的content 传递时 使用引号包围起来就可以

**如果传递的是非字符串类型 比如数字类型 比如布尔值 传递时 需要通过单大括号 比如下面的num**
```javascript
<FunctionComponent info={this.state} content='hello' num={17}>
  this is a function component !!
</FunctionComponent>
```
